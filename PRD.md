# 익명 칭찬 우체통 (Anonymous Compliment Mailbox)

## 1. 서비스 개요

온보딩 기간 동안 동기들에게 익명으로 칭찬과 응원 메시지를 전달하는 디지털 롤링페이퍼 서비스

### 핵심 컨셉
> "누가 보냈는지는 모르지만, 누군가 나를 응원하고 있다"

---

## 2. 기획 의도

### 배경
- 낯선 환경에서 온보딩 중인 신입 크루들이 서로에게 느낀 작은 고마움이나 응원을 부담 없이 표현할 창구 필요
- 직접 말하기 어색한 칭찬을 익명으로 전달하여 심리적 부담 감소
- 온보딩 기간 동안의 유대감 형성 및 긍정적인 팀 문화 조성

### 목표
- 동기들 간 따뜻한 메시지 교환을 통한 소속감 향상
- 간단하고 직관적인 UI로 누구나 쉽게 사용 가능
- GitHub Pages를 활용한 빠른 배포 및 접근성 확보

---

## 3. 사용자 시나리오 (User Flow)

### 3.1 입장
1. 서비스 접속
2. 온보딩 동기들의 이름(또는 닉네임)이 적힌 **우편함 리스트** 확인

### 3.2 메시지 작성
1. 칭찬하고 싶은 동기의 우편함 클릭
2. 익명 메시지 작성
3. 귀여운 스티커/이모지 선택
4. 메시지 전송

### 3.3 메시지 확인
1. 내 우편함 클릭
2. 동기들이 남긴 메시지 카드가 그리드 형태로 표시
3. 개별 카드 클릭 시 전체 메시지 확인

---

## 4. 주요 기능

### 4.1 우편함 리스트
- 모든 동기들의 우편함을 카드/리스트 형태로 표시
- 각 우편함에 받은 메시지 개수 표시 (선택사항)
- 검색/필터 기능 (인원이 많을 경우)

### 4.2 메시지 작성
- 텍스트 메시지 입력 (최대 200자)
- 스티커/이모지 선택 (8~12종)
- **편지지 디자인 선택** (6종 이상의 감성적인 편지지 템플릿)
  - 꽃무늬, 별무늬, 구름, 하트, 레트로, 심플 등
- **손글씨 폰트 옵션** (3종)
  - 기본 폰트, 귀여운 손글씨, 정갈한 손글씨
- **칭찬 템플릿** (빠른 문구 선택 기능)
  - "항상 밝은 에너지 고마워요!"
  - "발표 정말 잘했어요!"
  - "같이 일해서 즐거워요!"
  - "배려해줘서 감사해요!"
  - "앞으로도 파이팅!"
  - 직접 입력 옵션
- 익명 전송 (발신자 정보 저장 안함)

### 4.3 내 우편함
- 받은 메시지 그리드 뷰로 표시
- 메시지 카드 디자인 (스티커 + 메시지 미리보기)
- 카드 클릭 시 모달로 전체 내용 확인

### 4.4 전체 공개 칭찬판 (Live Praise Board)
- 헤더 영역에 실시간 애니메이션으로 표시되는 공개 칭찬 메시지
- **표시 형식**: "To. [수신자 이름] - [칭찬 메시지 일부]"
- **애니메이션 효과**:
  - 페이드인/페이드아웃으로 자연스럽게 전환
  - 5초 간격으로 다음 메시지로 로테이션
  - 슬라이드 또는 타이핑 효과 적용
- **데이터 소스**: 전송된 메시지 중 최근 메시지를 랜덤하게 노출
- **익명성 유지**: 발신자는 표시하지 않음, 수신자와 내용 일부만 공개
- 메시지가 없을 경우 기본 응원 문구 표시

### 4.5 부가 기능
- 새 메시지 알림 표시
- 메시지 총 개수 통계
- 반응형 디자인 (모바일 대응)

## 7. 데이터 구조

### 7.1 사용자 (Members)
```json
{
  "id": "member_1",
  "name": "민수",
  "avatar": "emoji_or_image_url"
}
```

### 7.2 메시지 (Messages)
```json
{
  "id": "msg_1",
  "recipientId": "member_1",
  "content": "온보딩 기간에 많이 도와줘서 고마워!",
  "sticker": "star",
  "paper": "flower",
  "font": "cute",
  "createdAt": "2024-01-15T09:00:00Z"
}
```

### 7.3 편지지 종류 (Papers)
| ID | 이름 | 설명 |
|---|---|---|
| flower | 꽃무늬 | 연분홍 배경에 꽃 패턴 |
| star | 별무늬 | 연보라 배경에 별 패턴 |
| cloud | 구름 | 하늘색 배경에 구름 패턴 |
| heart | 하트 | 연분홍 배경에 하트 패턴 |
| retro | 레트로 | 빈티지 느낌의 베이지 패턴 |
| simple | 심플 | 깔끔한 단색 배경 |

### 7.4 폰트 종류 (Fonts)
| ID | 이름 | 설명 |
|---|---|---|
| default | 기본 | 시스템 기본 폰트 |
| cute | 귀여운 손글씨 | 동글동글한 귀여운 느낌 |
| elegant | 정갈한 손글씨 | 깔끔하고 정돈된 느낌 |

### 7.5 칭찬 템플릿 (Templates)
```json
[
  { "id": 1, "text": "항상 밝은 에너지 고마워요!" },
  { "id": 2, "text": "발표 정말 잘했어요!" },
  { "id": 3, "text": "같이 일해서 즐거워요!" },
  { "id": 4, "text": "배려해줘서 감사해요!" },
  { "id": 5, "text": "앞으로도 파이팅!" }
]
```

## 8. 제약사항 및 고려사항

### 보안
- 익명성 보장 (발신자 정보 저장 안함)
- 부적절한 메시지 필터링 고려 (욕설 등)

### UX
- 메시지 최대 글자수 제한 (200자)
- 모바일 환경 최적화
- 직관적인 UI/UX

### 운영
- 멤버 리스트 관리 방법 (JSON 파일 또는 관리자 페이지)
- 서비스 오픈/마감 기간 설정 (선택사항)
